Examples

The examples, which are shown in this chapter of the documentation, provide a quick overview about how to use the library "AndroidMaterialPreferences". This includes the necessary setup steps, as well as source code examples, which demonstrate the use of the library.

Project setup

Before the library "AndroidMaterialPreferences" can be used, its source code must be retrieved. The latest release can be downloaded as a zip archive via the direct link https://sourceforge.net/p/androidmaterialpreferences/files. Alternatively, the Mercurial repository, which hosts the project's source code, can be cloned by using the following command line command:

hg clone http://hg.code.sf.net/p/androidmaterialpreferences/code androidmaterialpreferences-code

To use the library within an Android project, the downloaded source code has to be imported into the workspace of the appropriate IDE. When using Eclipse, the source code can be added as a new project by using the option "File" -> "Import..." -> "Android" -> "Existing Android Code Into Workspace".

To be able to use the newly imported library project from within another project, a reference to the library project has to be created. This can be established in the Eclipse IDE by right-clicking the dependent project and choosing the option "Properties". Then the group "Android" at the left of the recently opened window has to be chosen. Finally, the button "Add..." has to be clicked to add the library project as a new dependency. For further information about how to reference an Android library project, refer to this article of the official Android developer documentation.

As the library depends on the AppCompat v7 revision 21 support library in order to provide Android 5.0-features on devices with an API level less than 21, this support library or a newer one has to be added to the library's workspace as well. A detailed explaination about how to import this support library can be found in this article of the Android developer documentation.

Declaring a SeekBarPreference via XML

The XML code below shows how to declare a SeekBarPreference as a XML resource. The example contains all of the preference's custom attributes.

<?xml version="1.0" encoding="utf-8"?>
<PreferenceScreen xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:custom="http://schemas.android.com/apk/res-auto" >

    <de.mrapp.android.preference.SeekBarPreference
        android:defaultValue="33.336"
        android:dialogTitleColor="@android:color/black"
        android:dialogButtonTextColor="@android:color/black"
        android:dialogMessage="@string/seek_bar_preference_dialog_message"
        android:dialogTitle="@string/seek_bar_preference_dialog_title"
        android:key="@string/seek_bar_preference_key"
        android:max="150"
        android:summary="@array/seek_bar_preference_summaries"
        android:title="@string/seek_bar_preference_title"
        custom:decimals="2"
        custom:floatingPointSeparator="."
        custom:min="25"
        custom:showProgress="true"
        custom:showValueAsSummary="false"
        custom:suffix="Unit" />

</PreferenceScreen>

The value of the attribute android:summary can be set to the resource ID of a string or of a string array. All values, which are defined by using resource IDs in the example above, can also be defined by directly assigning the required value.

Instantiating a SeekBarPreference programatically

The Java code below shows how a SeekBarPreference can be initialized programmatically. The attributes, which are configured by using the preference's setter methods, are the same as shown in the previous example.

SeekBarPreference preference = new SeekBarPreference(this);
preference.setDefaultValue(33.336f);
preference.setDialogTitleColor(Color.BLACK);
preference.setDialogButtonTextColor(Color.BLACK);
preference.setDialogMessage(R.string.seek_bar_preference_dialog_message);
preference.setDialogTitle(R.string.seek_bar_preference_dialog_title);
preference.setKey(R.string.seek_bar_preference_key);
preference.setMaxValue(150);
preference.setSummaries(R.array.seek_bar_preference_summaries);
preference.setTitle(R.string.seek_bar_preference_title);
preference.setDecimals(2);
preference.setFloatingPointSeparator(".");
preference.setMinValue(25);
preference.showProgress(true);
preference.showValueAsSummary(false);
preference.setSuffix("Unit");

As seen above, the values of all String attributes can either be assigned by directly passing a String reference to the appropriate setter method, or by passing the resource ID of a String resource.

Declaring an EditTextPreference via XML

The example below shows how the attributes of an EditTextPreference can be specified when the preference is declared using a XML resource.

<?xml version="1.0" encoding="utf-8"?>
<PreferenceScreen xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:custom="http://schemas.android.com/apk/res-auto" >

    <de.mrapp.android.preference.EditTextPreference
        android:defaultValue="DefaultText"
        android:dialogTitleColor="@android:color/black"
        android:dialogButtonTextColor="@android:color/black"
        android:dialogMessage="@string/edit_text_preference_dialog_message"
        android:dialogTitle="@string/edit_text_preference_dialog_title"
        android:key="@string/edit_text_preference_key"
        android:summary="@string/edit_text_preference_summary"
        android:title="@string/edit_text_preference_title"
        custom:showValueAsSummary="false" />

</PreferenceScreen>

Instantiating an EditTextPreference programatically

The code below illustrates how to instantiate an EditTextPreference in a programmatically mannor. The preferences configuration is equal to the one, which has been shown in the previous example.

EditTextPreference preference = new EditTextPreference(this);
preference.setDefaultValue("DefaultText");
preference.setDialogTitleColor(Color.BLACK);
preference.setDialogButtonTextColor(Color.BLACK);
preference.setDialogMessage(R.string.edit_text_preference_dialog_message);
preference.setDialogTitle(R.string.edit_text_preference_dialog_title);
preference.setKey(R.string.edit_text_preference_key);
preference.setSummary(R.string.edit_text_preference_summary);
preference.setTitle(R.string.edit_text_preference_title);
preference.showValueAsSummary(false);

Declaring a ListPreference via XML

The following example illustrates how the attributes of a ListPreference can be specified, if the preference should be declared within a XML resource.

<?xml version="1.0" encoding="utf-8"?>
<PreferenceScreen xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:custom="http://schemas.android.com/apk/res-auto" >

    <de.mrapp.android.preference.ListPreference
        android:defaultValue="0"
        android:entries="R.array.list_preference_entries"
        android:entryValues="R.array.list_preference_entry_values"
        android:dialogTitleColor="@android:color/black"
        android:dialogButtonTextColor="@android:color/black"
        android:dialogMessage="@string/list_preference_dialog_message"
        android:dialogTitle="@string/list_preference_dialog_title"
        android:key="@string/list_preference_key"
        android:summary="@array/list_preference_summary"
        android:title="@string/list_preference_title"
        custom:showValueAsSummary="false" />

</PreferenceScreen>

Instantiating a ListPreference programatically

The following code shows how to programmatically instantiate a ListPreference.

ListPreference preference = new ListPreference(this);
preference.setDefaultValue("0");
preference.setEntries(R.array.list_preference_entries);
preference.setEntryValues(R.array.list_preference_entry_values);
preference.setDialogTitleColor(Color.BLACK);
preference.setDialogButtonTextColor(Color.BLACK);
preference.setDialogMessage(R.string.list_preference_dialog_message);
preference.setDialogTitle(R.string.list_preference_dialog_title);
preference.setKey(R.string.list_preference_key);
preference.setSummary(R.string.list_preference_summary);
preference.setTitle(R.string.list_preference_title);
preference.showValueAsSummary(false);
